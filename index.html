Loops
Loops let you repeat code. Instead of writing the same line 100 times, write it once and loop it. Two types: for and while.

for Loop — Loop a fixed number of times

python
# Loop through a range of numbers
for i in range(5):
    print(f"Count: {i}")

# Output: 0, 1, 2, 3, 4 (starts at 0!)

# Loop through a list
fruits = ["apple", "banana", "mango"]
for fruit in fruits:
    print(f"I love {fruit}")

# range(start, stop, step)
for i in range(1, 11, 2):  # 1,3,5,7,9
    print(i)

while Loop — Loop until condition is false

python
count = 0
while count < 5:
    print(f"Count is {count}")
    count += 1   # count = count + 1

# break — stop the loop early
for i in range(10):
    if i == 5:
        break   # stops at 5
    print(i)

# continue — skip this iteration
for i in range(10):
    if i % 2 == 0:
        continue   # skip even numbers
    print(i)   # prints: 1,3,5,7,9

Error Handling
Real programs break. Error handling lets your program fail gracefully instead of crashing. Professional code always handles errors — this is what separates beginners from real developers.

Real-World Error Handling — API Calls

python
import requests

def call_api(url):
    try:
        response = requests.get(url, timeout=10)
        response.raise_for_status()  # Raise error if 4xx/5xx
        return response.json()
    except requests.exceptions.Timeout:
        print("Request timed out")
    except requests.exceptions.HTTPError as e:
        print(f"HTTP Error: {e}")
    except Exception as e:
        print(f"Error: {e}")
    return None

Object-Oriented Programming lets you model real things in code. A class is a blueprint. An object is a real instance of that blueprint. FastAPI and most frameworks use this heavily.

Your First Class

class Employee:
    # __init__ runs when you create a new object

        self.role = role

🔵 Intermediate · Lesson 16
Lambda, Map & Filter
Lambda is a mini function in one line. map() and filter() apply functions to lists. These are used heavily in data processing and API response handling.
python
# Lambda — anonymous one-line function
square = lambda x: x ** 2
print(square(5))  # 25

add = lambda a, b: a + b
print(add(3, 7))  # 10

# map() — apply function to every item
numbers = [1, 2, 3, 4, 5]
doubled = list(map(lambda x: x * 2, numbers))
print(doubled)  # [2,4,6,8,10]

# filter() — keep only items that match
scores = [45, 80, 55, 90, 30, 75]
passing = list(filter(lambda x: x >= 60, scores))
print(passing)  # [80, 90, 75]

# sorted() with key — sort by custom criteria
students = [("Ranjan", 85), ("Priya", 92), ("Amit", 78)]
ranked = sorted(students, key=lambda x: x[1], reverse=True)
print(ranked)  # [('Priya',92), ('Ranjan',85), ('Amit',78)]

🟣 Advanced · Lesson 18
Decorators
Decorators modify functions without changing their code. FastAPI uses @app.get() and @app.post() — these are decorators. Understanding them is essential for backend development.
python
# A decorator is a function that wraps another function
def timer(func):
    import time
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} took {end-start:.4f}s")
        return result
    return wrapper

# Apply decorator with @
@timer
def slow_function():
    import time
    time.sleep(1)
    print("Done!")

slow_function()
# Done!
# slow_function took 1.0012s

# FastAPI uses decorators exactly like this:
# @app.get("/users")   ← this is a decorator
# def get_users():     ← this is the function it decorates
#     return users
🟣 Advanced · Lesson 19
Generators & yield
Generators produce values one at a time instead of all at once. They save massive amounts of memory when dealing with large datasets. Claude's streaming API uses generators.
python
# Normal function — loads everything in memory
def get_numbers(n):
    return [i for i in range(n)]  # all in memory

# Generator — produces one at a time
def gen_numbers(n):
    for i in range(n):
        yield i  # pause here and return i

# Use it
gen = gen_numbers(5)
print(next(gen))  # 0
print(next(gen))  # 1

for num in gen_numbers(1000000):
    print(num)  # memory-efficient — never stores all 1M numbers
    if num > 5: break

# Claude streaming uses this pattern
def stream_claude(prompt):
    with client.messages.stream(model="...", ...) as stream:
        for text in stream.text_stream():
            yield text  # stream each chunk

Async / Await
Async lets your program do multiple things at once without waiting. FastAPI is async by default. When calling the Claude API, your server can handle other requests while waiting for Claude's response.
python
import asyncio

# async function — can pause and let others run
async def fetch_data(name):
    print(f"Fetching {name}...")
    await asyncio.sleep(2)  # simulate API wait
    return f"{name} data ready"

# Run multiple async tasks simultaneously
async def main():
    results = await asyncio.gather(
        fetch_data("Users"),
        fetch_data("Reports"),
        fetch_data("Stats")
    )
    for r in results:
        print(r)

asyncio.run(main())
# All 3 run simultaneously — takes 2s, not 6s!

# FastAPI async endpoint
@app.get("/data")
async def get_data():
    result = await call_claude_api()  # non-blocking
    return {"result": result}
